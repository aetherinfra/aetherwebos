# Governance

This document describes how the Aether project is organized and how decisions are made.

Aether is an open-source systems research project focused on operating system development, networking infrastructure, and low-level architecture.

---

## Project Structure

The Aether ecosystem is maintained through the GitHub organization:

```bash
aetherinfra
```

Repositories within the organization may include both public and private projects related to the Aether platform.

---

## Core Maintainers

The project is managed by the **Core Maintainers** team.

The `core-maintainers` team within the GitHub organization currently holds **merge authority** over the codebase.

Responsibilities include:

- Reviewing and merging pull requests
- Maintaining overall code quality
- Ensuring architectural consistency
- Managing issues and discussions
- Coordinating community contributions
- Maintaining project documentation
- Organizing contributor recognition and certificates

Core maintainers do not necessarily write all code themselves, but they ensure that contributions are properly reviewed and integrated.

## Contribution Process

Development primarily occurs through the standard GitHub workflow:

1. A contributor forks the repository
2. A feature branch is created
3. Changes are implemented
4. A pull request is opened
5. A core maintainer reviews the change
6. The pull request is merged if approved

Contributors are encouraged to check the **Issues** tab for open tasks and areas where help is needed.

---

## Decision Making

Technical decisions are generally made through discussion in:

- GitHub issues
- pull request reviews
- maintainer discussions

Core maintainers guide architectural decisions and ensure long-term project consistency.

---

## Community

Aether is an open-source project and welcomes contributions from developers interested in:

- operating systems
- networking stacks
- hardware drivers
- ARM architecture
- systems programming

Community participation is essential to the growth of the project.

---

## Evolution of Governance

The governance structure may evolve as the project grows and new contributors become maintainers.

Future roles may include:

- subsystem maintainers
- documentation maintainers
- release managers

Changes to governance will be documented in this file.

---

Aether is built collaboratively and guided by its core maintainers, with the goal of creating a strong open systems research platform.
